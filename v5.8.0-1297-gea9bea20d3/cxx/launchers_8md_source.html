1 # Launcher arguments for using mesa
2 
3 On linux, `paraview_mesa` launcher has now been removed. ParaView
4 binaries now respect command line arguments instead to indicate if Mesa3D
5 libraries should be loaded for software rendering, instead of using OpenGL
6 libraries available on your system. Pass `--mesa` with optional `--backend
7 [swr|llvm]` to any of the ParaView executables to request Mesa3D-based rendering
8 with approrpiate backend.

11 # Using MPICH ABI compatible implementation
12 
13 ParaView binaries can now be used on linux systems with MPICH ABI compatible
14 MPI implementation. Compatible MPI implementations are listed
15 [here](https://wiki.mpich.org/mpich/index.php/ABI_Compatibility_Initiative). To
16 use system MPI implementation instead of the one packaged with ParaView, pass
17 `--system-mpi` command line argument to any of the ParaView executables.

1 # Choosing DISPLAY or EGL device using command line arguments
2 
3 When multiple GPUs are available on a system, users have to set either the
4 DISPLAY environment variable or the EGL device index for each ParaView rank
5 correctly to ensure that a rank uses a specific GPU. To make this easier, we
6 now have two new command line arguments added to pvserver, pvrendering and
7 pvbatch executables.
8 
9 `--displays=` can be used to sepecify a comma separated list of available
10 display names, for example:
11 
12  > .. pvserver --displays=:0,:1,:2
13  > .. pvserver --displays=host1:0,host2:0
14 
15 For EGL, these can be EGL device index, e.g.
16 
17  > .. pvserver --displays=0,1
18 
19 These displays (or devices) are then assigned to each of the ranks in a
20 round-robin fashion. Thus, if there are 5 ranks and 2 displays, the displays are
21 assigned as `0, 1, 0, 1, 0` sequentially for the 5 ranks.
22 
23 `--displays-assignment-mode=` argument can be used to customize this
24 default assignment mode to use a contiguous assigment instead. Accepted values
25 are 'contiguous' and 'round-robin'. If contiguous mode is used
26 for the 5 ranks and 2 displays example, the displays are assigned as
27 `0, 0, 0, 1, 1`.

1 ## Executing Python packages in a zip archive
2 
3 Python supports the ability to execute a Python package similar to standard
4 `.py` script. A `__main__.py` file inside the package is treated as the entry
5 point for such an invocation.
6 
7 It's not uncommon to bundle a Python package (or module) in a zip archive. To import a package
8 inside a zip archive, the standard approach is to add the zip archive to the
9 Python module search path (i.e. `sys.path`) and then import the package using
10 its name.
11 
12 To make this process a little easier, ParaView's Python executables i.e. pvbatch
13 and pvpython now support passing a zip archive on the command line instead of a
14 `.py` script. In that case, these executables will internally add the zip
15 archive to the Python module search path `sys.path` and attempt to load the
16 package with same name as the zip archive (without the extension).
17 
18 For example:
19 
20 ```
21 > pvpython /tmp/sample.zip
22 ```
23 
24 This is same as the following:
25 
26 ```
27 > export PYTHONPATH=/tmp/sample.zip:$PYTHONPATH
28 > pvpython -m sample
29 ```

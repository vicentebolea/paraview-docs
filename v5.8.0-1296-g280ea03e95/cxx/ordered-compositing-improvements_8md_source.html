1 ## Improvements to ordered compositing
2 
3 Ordered compositing is used whenever ParaView is rendering
4 translucent geometry or volumetric data in parallel. For ordered
5 compositing, the data in the scene needs to be redistributed across
6 all rendering ranks to ensure the ranks has non-overlapping datasets with
7 other ranks. The data redistribution code has been revamped. It now
8 internally uses the `vtkRedistributeDataSet` filter. This greatly improves
9 performance when volume rendering unstructured grids in parallel. The code is also
10 much simpler. However, this introduces non-backwards compatible API changes that may
11 impact certain custom representations. Please refer to the `MajorAPIChanged` document
12 for details.

1 ## Changes to Cinema support
2 
3 Extract Generators introduced in this release also impact support for generation
4 of Cinema databases. Cinema Science project has undergone several changes over
5 the years with latest specification called Spec D, or just Cinema specification
6 for short. In this release, we simply ability to export Cinema specification by
7 integrating it with extract generators.
8 
9 Image Extract Generators such as PNG, JPEG now support generation of images from
10 multiple camera angles in addition the one used during the visualization setup.
11 By setting the **Camera Mode** to **Phi-Theta** on the **Properties** panel for
12 the extract generator, you can make the extract generator save out multiple
13 images per timestep, each done from a different camera position along a sphere
14 centered at the focal point with a radius set to the focal length.
15 
16 Tools provided by the Cinema Science project are intended to be used to explore
17 artifacts produced by analysis apps like ParaView. To use those tools, you have
18 to generate a summary database for all generated extracts. To generate this
19 summary, simply check **Generate Cinema Specification** checkbox either when
20 using **Save Extracts** to generate extracts immediately in application or
21 when using **Save Catalyst State** to generate extracts in situ.
22 
23 Cinema Science project is developing a new format for defining a composable
24 image set. This will support the ability of recoloring rendering results, and
25 combining multiple layers. Support for these composable-image-sets (CIS) will
26 be added in the future as a new type of Image Extract Generator.
27 
28 Also removed in this release is the ability to import Cinema databases. The
29 importer was designed for legacy Cinema specifications and hence removed.

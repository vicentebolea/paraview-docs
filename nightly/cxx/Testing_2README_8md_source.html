1 Baseline images
2 ===============
3 
4 ParaView has a large number of tests that involve executing some ParaView
5 functionality and then checking that the resulting image in the RenderView
6 is the same as some reference baseline image to within some tolerance. These
7 baseline images are stored in this directory under `Data/Baseline`. In many
8 cases, there is one baseline image per test. However, some tests generate more
9 than one output image. Each output has a distinct baseline image against which
10 it is compared.
11 
12 
13 Alternate baselines
14 -------------------
15 
16 Some tests produce slightly different results on different platforms.
17 To accommodate these differences, ParaView's testing infrastructure
18 supports alternate baseline images. Naming alternate baseline images
19 follows a simple pattern involving adding an underscore followed by a
20 sequentially increasing integer to the end of the test name. For instance,
21 for a test named `MyTest`, the primary and alternate baselines could be:
22 
23 * `MyTest.png`
24 * `MyTest_1.png`
25 * `MyTest_2.png`
26 
27 Unfortunately, this naming scheme does not provide any information
28 about which dashboard configuration requires the alternate
29 baseline. This file is intended to serve as a map between alternate
30 baseline name and the dashboard configuration that requires it. This
31 information can be used to easily generate new alternate baselines
32 with the appropriate number by mapping the dashboard configuration
33 back to the alternate baseline number.
34 
35 Note that one alternate baseline may be appropriate for more than one
36 dashboard configuration. In such cases, only one dashboard configuration
37 is listed as it should be sufficient to generate the alternate baseline
38 for that one machine.

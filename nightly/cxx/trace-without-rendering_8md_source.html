1 # Python trace without rendering components
2 
3 Python trace generation is an often used capability to understand
4 how to script ParaView pipelines using Python. These traces, however, can
5 get quite verbose. If you are not interested in the rendering parts of the script
6 and are only using the trace to setup the data processing pipeline, it is handy to not
7 including any code for the rendering components in the generated trace to minimize complexity.
8 To support that, we have added an ability to skip all rendering components including views, representations, color transfer functions, scalar bar etc. when generating Python trace or
9 saving Python state files. A checkbox, **Skip Rendering Components**, is now available on
10 the Python trace (or state) options dialog that you can check to skip all rendering components.

1 # Command line arguments and ParaViewPython executables
2 
3 A standard Python executable has several command line options arguments that often used
4 when running Python scripts. ParaView's python executables like `pvbatch` and
5 `pvpython` which are often used to run ParaView Python scripts. Traditionally,
6 however, these ParaView python executables have provided no means of passing
7 arguments to the Python interpreter itself. We address this issue with these
8 changes.
9 
10 Now, both `pvpython` and `pvbatch` pass all unknown arguments to the Python
11 interpreter. Thus, to simply run the demo script from the `paraview.demos.demo1`
12 module, one can do the following:
13 
14 ```bash
15 > ./bin/pvpython -m paraview.demos.demo1
16 
17 ```
18 
19 You can also explicit pass all subsequent arguments to the Python interpreter
20 using the `--` separator.
21 
22 For example:
23 
24 ```bash
25 # here, --help is parsed and interpretted by ParaView
26 > ./bin/pvpython --help
27 
28  --connect-id=opt Set the ID of the server and client to make sure they match.
29  0 is reserved to imply none specified.
30  --cslog=opt ClientServerStream log file.
31  ...
32 
33 # here, --help is instead processed by the Python interpreter used by ParaView
34 > ./bin/pvpython -- --help
35 usage: ./bin/pvpython [option] ... [-c cmd | -m mod | file | -] [arg] ...
36 Options and arguments (and corresponding environment variables):
37 -b : issue warnings about str(bytes_instance), str(bytearray_instance)
38  and comparing bytes/bytearray with str. (-bb: issue errors)
39 ...
40 
41 ```

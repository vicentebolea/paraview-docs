1 Proxy Hints And Annotations {#ProxyHints}
2 ===========================
3 
4 This page documents *Proxy Hints*, which are XML tags accepted under *Hints*
5 for a *Proxy* element in the Server-Manager configuration XMLs.
6 
7 WarnOnRepresentationChange
8 --------------------------
9 Warn the user on changing to a specific representation type.
10 
11 For the motivation behind this hint, see BUG #15117.
12 This is used to indicate to the pqDisplayRepresentationWidget that the user must
13 be prompted with a *'Are you sure?'* if they manually switch to this
14 representation from the UI.
15 
16  <RepresentationProxy ...>
17  ...
18  <Hints>
19  <WarnOnRepresentationChange value="Volume" />
20  </Hints>
21  </RepresentationProxy>
22 
23 WarnOnCreate
24 ------------
25 Warn the user when creating the filter or source in the UI.
26 
27 The motivation behind this hint is to warn the user when executing filters
28 like **Temporal Statistics** filter since they can potentially take a long time
29 for large file series.
30 
31  <SourceProxy ...>
32  ...
33  <Hints>
34  <WarnOnCreate title="Potentially slow operations">
35  **Temporal Statistics** filter needs to process all timesteps
36  available in your dataset and can potentially take a long time to complete.
37  Do you want to continue?
38  </WarnOnCreate>
39  </SourceProxy>

41 ReaderFactory
42 -------------
43 Mark a proxy as reader proxy so that it's used to open files from the **File |
44 Open** dialog.
45 
46 This hint is used to mark a proxy as a reader. It provides the ParaView
47 application with information about extensions supported by this reader.
48 **extensions** attribute to list the supported extensions e.g. "foo foo.bar" for
49 files named as `somename.foo` or `somename.foo.bar`.
50 **filename_patterns** attribute is used to list the filename patterns to match.
51 The format is similar to what one would use for `ls` or `dir` using wildcards e.g.
52 spcth\* to match spcta, spctb etc.
53 
54  <!-- using extensions -->
55  <SourceProxy ...>
56  ...
57  <Hints>
58  <ReaderFactory extensions="[space separated extensions w/o leading '.']"
59  filename_patterns="[space separated filename patters (using wildcards)]"
60  file_description="[user-friendly description]" />
61  </Hints>
62  </SourceProxy>

64 View
65 ----
66 Specify the default view to use for showing the output produced by a
67 source/filter.
68 
69 This hint is used to indicate the name of the view to use by default for showing
70 the output of this source/filter on first *Apply*. To specify the view type for
71 a specific output port, you can use the optional attribute **port**.
72 
73  <SourceProxy ...>
74  ...
75  <Hints>
76  <View type="XYChartView" />
77  </Hints>
78  </SourceProxy>
79 
80 In certain cases, in addition to showing the data in the "preferred" default
81 view, you may want to show the result in the current view as well e.g. when
82 **Plot Over Line** filter is created, while the result is shown in the
83 **XYChartView**, we want to show the line representation for the data being
84 plotted in the current Render View too. For that one can add the
85 `also_show_in_current_view` attribute to the `<View/>` tag.
86 
87  <SourceProxy ...>
88  ...
89  <Hints>
90  <View type="XYChartView" also_show_in_current_view="1" />
91  </Hints>
92  </SourceProxy>
93 
94 If the source/filter has more than 1 output ports, you can choose which port the
95 hint corresponds to by using the optional `port` attribute.
96 
97  <SourceProxy ...>
98  ...
99  <Hints>
100  <View type="XYChartView" port="1" />
101  </Hints>
102  </SourceProxy>
103 
104 The `<View/>` hint can also be used to prevent automatic display of an output in
105 any view. To do that, use the **None** type. Such source can then still be
106 displayed manually by toggling the visibility when an appropriate View is active.
107 
108  <SourceProxy ...>
109  ...
110  <Hints>
111  <View type="None" />
112  </Hints>
113  </SourceProxy>

115 PipelineIcon
116 ------------
117 Specify the pipeline icon view to use in the pipeline browser for this
118 source/filter proxy.
119 
120 This hint is used to indicate the icon to use in the pipeline browser.
121 It can be either the full name of a qt resource icon or the name of a view type
122 for which an icon as already been defined.
123 
124  <SourceProxy ...>
125  ...
126  <Hints>
127  <PipelineIcon name="XYChartView" />
128  </Hints>
129  </SourceProxy>
130 
131 If the source/filter has more than 1 output port, you can choose which port the
132 hint corresponds to by using the optional `port` attribute.
133 
134  <SourceProxy ...>
135  ...
136  <Hints>
137  <PipelineIcon name="XYChartView" port="1" />
138  </Hints>
139  </SourceProxy>
140 
141 At the time of writing, the supported stock icon names were:
142  * "SERVER"
143  * "SECURE_SERVER"
144  * "LINK"
145  * "GEOMETRY"
146  * "XYChartView"
147  * "XYBarChartView"
148  * "XYHistogramChartView"
149  * "BoxChartView"
150  * "SpreadSheetView"
151  * "INDETERMINATE"
152  * "None"
153  * "EYEBALL"
154  * "EYEBALL_GRAY"
155  * "INSITU_EXTRACT"
156  * "INSITU_EXTRACT_GRAY"
157  * "INSITU_SERVER_RUNNING"
158  * "INSITU_SERVER_PAUSED"
159  * "INSITU_BREAKPOINT"
160  * "INSITU_WRITER_PARAMETERS"
161  * "CINEMA_MARK"
162 
163 If the desired icon is not present in the list, it is possible to use a Qt resource icon name directly.
164 
165  <SourceProxy ...>
166  ...
167  <Hints>
168  <PipelineIcon name=":/pqWidgets/Icons/pqCalculator.svg" />
169  </Hints>
170  </SourceProxy>
171 
172 Available icons are visible in the sources of ParaView
173 
174 If the desired icon is not present, it can be added, for example in the context of a plugin, using
175 GUI_RESOURCES in your ADD_PARAVIEW_PLUGIN macro, a .qrc file and your own icon file.
176 
177  <SourceProxy ...>
178  ...
179  <Hints>
180  <PipelineIcon name=":/MyPluginQtResource/Icons/myIcon.png" />
181  </Hints>
182  </SourceProxy>

184 Plotable
185 --------
186 Mark output data as plotable in 2D chart views.
187 
188 Chart views in ParaView e.g. **Bar Chart View**, **Line Chart View**, support
189 plotting data of any type. However, since such plots don't use distributed
190 rendering techniques, to avoid accidentally plotting large datasets, the plots
191 by default can only show sources/filters that produce `vtkTable` as the output.
192 If a source/filter doesn't produce a `vtkTable`, but produces data that should
193 indeed be plotted by such views, one can use this hint.
194 
195  <SourceProxy ...>
196  <Hints>
197  <Plotable />
198  </Hints>
199  </SourceProxy>

201 RepresentationType
202 ------------------
203 Specify the representation type to use by default when showing the output from a
204 source/filter in a particular view.
205 
206 This hint is used to indicate the default representation type in any/all views.
207 The **view** attribute is optional. When not specified it matches all views.
208 Likewise, **port** attribute is optional. When not specified it matches all
209 output ports. The hints are processed in order. Hence when specifying multiple
210 Representation elements, start with most restrictive to least restrictive.
211 
212 Note, this hint doesn't control which representation proxy gets created, but the
213 default value for the "Representation" property on the representation proxy
214 set using `vtkSMRepresentationProxy::SetRepresentationType()`.
215 
216  <SourceProxy ...>
217  ...
218  <Hints>
219  <RepresentationType view="ComparativeRenderView" type="Surface" port="1"/>
220  <RepresentationType view="RenderView" type="Wireframe" />
221  </Hints>
222  </SourceProxy>
223 
224 Representation
225 --------------
226 Specify the representation proxy to create to show the output from a
227 source/filter in a particular view.
228 
229 This hint is used to indicate the representation proxy to create to show the
230 output from a source/filter in a particular view, rather than letting the view
231 determine which representation proxy to create. This is rare. The more common
232 use-case of picking the default representation type is satisfied by
233 **RepresentationType** XML hint documented above.
234 
235 The required **view** attribute specifies the view to which the hint applies and
236 should be set to the XML proxy name of the view. The required **type** attribute
237 specifies the XML proxy name for representation to create. The optional
238 **port** attribute can be used to limit the hint to specific output port.
239 
240  <SourceProxy ...>
241  ...
242  <Hints>
243  <Representation view="RenderView" type="TextSourceRepresentation" />
244  </Hints>
245  </SourceProxy>

247 OutputPort
248 --------------------------
249 Set name and representation of a specific output port.
250 
251 This hint is used to rename a specific output port
252 and also the type of representation to use.
253 Type can be either "text", "logo" or "progress".
254 
255 The representation part of this hint may be deprecated soon.
256 Use Representation hint instead.
257 
258  <SourceProxy ...>
259  ...
260  <Hints>
261  <OutputPort index="0"
262  name="Output-0"
263  type="text" />
264  </Hints>
265  </SourceProxy>

267 ShowProxyDocumentationInPanel
268 -----------------------------
269 Show an annotation label in the auto-generated panel generated using
270 pqProxyWidget.
271 
272 This hint is used to indicate that the documentation for the proxy should be
273 shown in special label at the top of the panel generated for the proxy. This is
274 useful to show information to the user directly on the panel.
275 
276 The ShowProxyDocumentationInPanel take one optional attribute **type**. The
277 possible values are:
278 1. *description*: (default) to use vtkSMDocumentation::GetDescription(),
279 2. *short_help*: to use vtkSMDocumentation::GetShortHelp(), and
280 3. *long_help*: to use vtkSMDocumentation::GetLongHelp().
281 
282  <SourceProxy ...>
283  <Documentation>
284  Some text that will be shown in the label.
285  </Documentation>
286  ...
287  <Hints>
288  <ShowProxyDocumentationInPanel type="description"/>
289  </Hints>
290  </SourceProxy>

292 ReloadFiles
293 -----------
294 Indicate the property on a reader to use to refresh (or reload) the reader to
295 make it re-read the data files.
296 
297 This hint can be used for readers that support "smart refresh" to re-read files
298 when they are changed. Otherwise, ParaView will use the default mechanism which
299 simply recreated the reader thus forgoing any previous data cached by the
300 reader. The attribute **property** indicates the name of the property on the
301 reader proxy to invoke to make the reader refresh.
302 
303  <SourceProxy>
304  ...
305  <Hints>
306  <ReloadFiles property="Refresh" />
307  </Hints>
308  </SourceProxy>

310 View Annotations
311 ----------------
312 Views support the following annotations:
313 
314 1. **ParaView::DetachedFromLayout**: **Deprecated in ParaView 5.7**.
315 This is no longer applicable as all views are created detached from layout by default.
316 
317 Live Source
318 ------------
319 Certain algorithms can generate new data autonomously, e.g. a source that reads
320 data from the network. The **LiveSource** hint allows ParaView to periodically
321 check with the algorithm if it has new data and update the application, if so.
322 
323 For that, one simply adds a hint to the proxy as follows:
324 
325  <SourceProxy ...>
326  ...
327  <Hints>
328  <LiveSource interval="100" />
329  </Hints>
330  </SourceProxy>
331 
332 The algorithm subclass must have `bool GetNeedsUpdate()` method that returns
333 true if the algorithm needs update.
334 
335 The `interval` attribute is optional (defaults to 100) and can be used to
336 provide a refresh rate in milliseconds.
337 
338 ConnectToPortIndex
339 --------------------------
340 Connect to a specific port index.
341 
342 This is used to connect a representation to an output port of a filter
343 other than the default (index = 0). Currently, this is only used to
344 modify the input port for the selection representation subproxy in
345 vtkSMPVRepresentationProxy.
346 
347  <RepresentationProxy ...>
348  ...
349  <Hints>
350  <ConnectToPortIndex value="2" />
351  </Hints>
352  </RepresentationProxy>

354 ShowInMenu
355 --------------------------
356 Plugin specific hint to control how a filter/source is shown in menus
357 
358 This is used **in plugins only** to configure how a source or filter
359 should be shown in the filters/sources menu. It enables to place them
360 into an existing category or a new category, as well as set the icon.
361 
362  <SourceProxy ...>
363  ...
364  <Hints>
365  <ShowInMenu category="Category" icon=":/path/to/ressource/icon.png" />
366  </Hints>
367  </SourceProxy>
